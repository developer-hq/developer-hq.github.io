jupyter notebook切换虚拟环境

https://blog.csdn.net/hu_wei123/article/details/127390535

  • 先在base和虚拟环境中安装ipykernel

  • 如果还没有虚拟环境:

    这时，我们 “将计就计” ，就认为是第 3 步出错，我们需要在此虚拟环境中继续执行以下命令，让虚拟环境名称在 Kernel，Change kernel 显示

    python -m ipykernel install --user --name 虚拟环境名 --display-name "Python (虚拟环境名)"

    cmd终端里安装 ipykernel ，我们看似已经成功安装，但是执行以上命令时仍然报错： 找不到ipykernel 模块

  • eg:python -m ipykernel install --user --name d2l

jupyter notebook 查看、添加、删除kernel的方法

https://blog.csdn.net/crystal_sugar/article/details/126855682

安装完Anaconda利用conda创建了虚拟环境，但是启动jupyter notebook之后却找不到虚拟环境。

实际上是由于在虚拟环境下缺少kernel.json文件，解决方法如下：

首先安装ipykernel：conda install ipykernel

在虚拟环境下创建kernel文件：conda install -n 环境名称 ipykernel

激活conda环境： source activate 环境名称

将环境写入notebook的kernel中

python -m ipykernel install –user –name 环境名称 –display-name “Python (环境名称)”

打开notebook服务器：jupyter notebook

浏览器打开对应地址，新建python，就会有对应的环境提示了

如果经常需要用jupyter notebook，那么最好在创建虚拟环境的时候便安装好ipykernel，

命令：conda create -n 环境名称 python=3.5 ipykernel

删除kernel环境：

jupyter kernelspec remove 环境名称

查看kernel环境：

jupyter kernelspec list

迁移学习

Posted on 2023-03-30

调用模型

可以直接使用model_ft.fc调用其中某层，并可以直接修改该层结构

官网示例

下载后的模型会保存在用户/.cache/torch/checkpoints下

  • model_ft = models.resnet152(pretrained=use_pretrained)

  • pretrained用不用人家训练好的数据

  • num_ftrs = model_ft.fc.in_features
    model_ft.fc = nn.Sequential(nn.Linear(num_ftrs, 102),nn.LogSoftmax(dim=1))

    • 直接修改其中某层，将fc层重新做成全连接层，输入为原输入2048，输出结果为102

  • set_parameter_requires_grad(model_ft, feature_extract)

    • 设置是否使用梯度，用梯度就代表着还要训练模型参数，不用梯度就代表着直接使用别人的参数，就不再训练了

一般先训练自己新建的层，然后再整体训练一下

是否冻住某些层

关闭梯度就属于冻住了

google搜索

Posted on 2023-03-28
  1. 使用引号精确搜索（“”）
  2. 使用连字符排除内容（-）
  3. 使用星号填充内容（*）
  4. 搜索指定范围（….）JavaScript in 2021….2022
  5. 搜索指定站点内容（search:）search: bugstack.cn spring
  6. 搜索指定文件类型内容（filetype）Java filetype:pdf
  7. 查找多个关键字（AND）Java AND C++
  8. 查找其中一个关键字（OR）Java OR C++
  9. 查询关键字的定义（Define）Define: Java
  10. 搜索相关内容（+）Java jvm+jdk
  11. 查找相似网站（related）relate: bugstack.cn
  12. 搜索指定站点内容（site）site: bugstack asm
  13. 搜索指定时间范围（before、after）after: 2022 jvm java
